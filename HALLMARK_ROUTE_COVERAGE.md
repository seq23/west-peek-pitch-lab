# Hallmark Route and State Coverage — Pitch Lab

A route-only screenshot is invalid. Every required state needs desktop and mobile evidence, final URL, visible controls, overflow/geometry result, console/network summary, and honest unproven boundaries.

| Route | Required states |
|---|---|
| `/` | first viewport, CTA geometry, mobile menu closed/open, Scooter welcome stage, final CTA |
| `/how-it-works` | first viewport, complete-flow section, mobile menu |
| `/practice` | profile, validation error, deck choice, first prompt, first-answer draft reveal, mid-session, completed state, Scooter expanded/compact, mobile draft sheet open/closed |
| `/story-card` | local card, provider unavailable, AI result, final Scooter summary state, Practice Out Loud idle/permission/recorded states |
| `/share` | blocked early share, populated preview, consent unchecked/checked, pending, confirmed, failed |
| `/thank-you` | no receipt, confirmed receipt |
| policy/support routes | desktop/mobile readability, footer navigation, long-content overflow |

Valid verdicts: APPROVED, APPROVED WITH WARNINGS, REDESIGN REQUIRED, INCOMPLETE — STATE COVERAGE MISSING, BLOCKED — ENVIRONMENT OR DEPLOYMENT ACCESS FAILED.
